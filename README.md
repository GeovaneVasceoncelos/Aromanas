## Descrição do caso escolhido

O projeto foi desenvolvido para a Aromanas, uma pequena loja especializada em produtos de higiene pessoal e aromatização de ambientes. A empresa comercializa produtos como sabonetes líquidos, cremes hidratantes, home sprays, aromatizadores e itens voltados ao bem-estar e conforto dos clientes.

A escolha da Aromanas foi motivada pelo fato de ser um pequeno negócio real que ainda não possui um site próprio, tornando o projeto uma oportunidade de aplicar os conhecimentos adquiridos na disciplina para solucionar uma necessidade concreta.

## Necessidade identificada

A Aromanas não possui presença digital por meio de um site próprio, o que dificulta a divulgação de seus produtos e o alcance de novos clientes. Atualmente, os consumidores dependem principalmente das redes sociais e do contato direto para conhecer os produtos oferecidos.

Dessa forma, identificou-se a necessidade de desenvolver uma página institucional que apresente a empresa, seus produtos e formas de contato de maneira organizada e acessível.

## Público-alvo

O público-alvo do site é composto por:

- Pessoas interessadas em produtos de higiene pessoal;
- Clientes que buscam aromatizadores para residências ou ambientes comerciais;
- Consumidores que valorizam produtos relacionados ao bem-estar e autocuidado;
- Pessoas que procuram opções de presentes e kits personalizados.

Esse público precisa encontrar facilmente informações sobre os produtos, a proposta da empresa, seus diferenciais e os meios de contato disponíveis.

## Objetivo do site

O principal objetivo do site é apresentar a Aromanas ao público, divulgar seus produtos e facilitar o contato entre clientes e a empresa.

A principal ação esperada do visitante é conhecer os produtos disponíveis e entrar em contato com a loja para obter mais informações ou realizar pedidos.

## Processo de desenvolvimento

O desenvolvimento do projeto foi realizado em etapas.

Inicialmente, a dupla realizou a análise do negócio escolhido e identificou as principais necessidades que poderiam ser atendidas por meio de um site institucional.

Em seguida, foi planejada a estrutura da página, definindo as seções que seriam necessárias para apresentar a empresa de forma clara e organizada. Após o planejamento, foi desenvolvido o HTML da página, seguido pela estilização utilizando CSS e Bootstrap.

Durante o processo, algumas alterações foram realizadas em relação à ideia inicial, principalmente na identidade visual. As cores foram ajustadas para seguir o padrão da logomarca da Aromanas, criando uma aparência mais harmoniosa e profissional. Também foram realizadas melhorias na organização dos cards de produtos, na seção de perguntas frequentes e no formulário de contato.

Por fim, foram implementadas as funcionalidades em JavaScript e realizados testes para garantir o funcionamento correto do site.

## Principais decisões do projeto

As principais decisões tomadas durante o desenvolvimento foram:

- Utilização de HTML semântico para melhorar a organização e acessibilidade do conteúdo;
- Estruturação da página em seções distintas para facilitar a navegação;
- Uso de uma paleta de cores inspirada na identidade visual da marca Aromanas;
- Desenvolvimento de um layout responsivo para diferentes tamanhos de tela;
- Utilização de imagens ilustrativas para destacar os produtos;
- Inclusão de textos alternativos (alt) nas imagens para acessibilidade básica;
- Implementação de elementos interativos para melhorar a experiência do usuário.

## Funcionalidade em JavaScript

Foram implementadas duas funcionalidades utilizando JavaScript.

A primeira está presente na seção de Perguntas Frequentes (FAQ), onde o usuário pode clicar em uma pergunta para exibir ou ocultar sua respectiva resposta. Essa funcionalidade foi escolhida para organizar melhor as informações e evitar excesso de conteúdo visível ao mesmo tempo.

A segunda funcionalidade foi implementada no formulário de contato. Após o preenchimento e envio dos dados, uma janela modal é exibida informando que a mensagem foi enviada com sucesso. Além disso, foi realizada uma validação simples para verificar se todos os campos obrigatórios foram preenchidos.

Essas funcionalidades foram escolhidas por serem úteis ao usuário e demonstrarem a integração entre HTML, CSS e JavaScript.

## Uso de Bootstrap

O Bootstrap foi utilizado para agilizar o desenvolvimento da interface e garantir maior responsividade.

Os principais recursos utilizados foram:

- Navbar responsiva;
- Sistema de Grid;
- Containers;
- Cards para exibição dos produtos;
- Botões estilizados;
- Formulários;
- Modal de confirmação.

A utilização do Bootstrap permitiu criar uma interface organizada, moderna e adaptável a diferentes dispositivos.

## Testes realizados

Foram realizados diversos testes durante o desenvolvimento do projeto, incluindo:

- Verificação da responsividade em diferentes tamanhos de tela;
- Testes dos links de navegação interna;
- Verificação do carregamento correto das imagens;
- Testes da funcionalidade das perguntas frequentes;
- Testes do formulário de contato e da janela modal;
- Verificação da acessibilidade básica por meio de textos alternativos em imagens e uso adequado de elementos semânticos;
- Testes da publicação e funcionamento geral do site.

Todos os testes apresentaram resultados satisfatórios após os ajustes realizados.
